Lesson 3 Highlights

Rose Pennington

Highlight #1 "Git"

"A free and open source (similar to Linux) distributed version control system is Git. It is designed to work with both small and large projects and assists with speed and efficiency. Git is specifically a distributed revision control system which works hard to increase speed, data integrity and supports non-linear distributed workflows. Git works on Linux, Windows and Mac Systems. Git commands are short, simple and to the point, similar to Unix"

Highlight #2 "Version Control"

"Version Control is a software tool. It allows developers to store of multiple versions (past and present) of their works (such as websites, documents and programs.) It makes it easy to resort back to a simpler model if a newer model has too many bugs or issues. It is also extremely useful for multiple developers to work on a single project. For example, say a new developer takes over for an older one, they can use version control to see past works to get a feel for what the previous developer was working towards."

Highlight #3 "GitHub"

"GitHub is an online webhost company geared for projects using Git. It is built for free public open-source code, and embodies the sharing programmer environment making a safe environment for new programmers to learn from more experienced. It also makes for a good accessible project portfolio for future employers or coworkers to see past works. GitHub also includes in-line editing, wikis and ticketing for its users. It is one of the most popular and common visual control websites due to it’s simplicity, design and user accessibility. Many companies and schools use it, both big and small, private and corporate."

Highlight #4 "Git Commands"

"Git Log shows all the backups created in the repository. Git Status shows all the files that were changed before the last backup and which are already in the staging area. Git blame shows who wrote which line of code, and therefore who can be blamed if that line of code goes wrong. Git diff shows the difference between the working directory and the backups which were created in the repository."

Highlight #5 "Repository"

"Git stores information in a repository, a type of data structure. A set of commit objects, a set of references to commit objects (called heads) are all stored in a git repository. You can get a Git project from cloning an existing Git repository from another server. Backups can be created in the repository and can be shown using ‘git log’ and a user can see the difference between those backups and the working directory using ‘git diff.’"
